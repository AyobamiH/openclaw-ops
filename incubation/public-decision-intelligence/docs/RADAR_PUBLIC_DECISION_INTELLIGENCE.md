# Strategic Radar

## Opportunity

Most public evidence releases are too large and fragmented for the average user
to interpret. A backend that reconstructs decisions rather than merely
summarizing documents creates a defensible capability with value for journalism,
research, oversight, and civic understanding.

## Positioning

The product should be positioned as:

- a decision intelligence platform
- an evidence-backed retrieval and reconstruction engine
- a living archive for public evidence bundles

It should not be positioned as:

- a chatbot
- a PDF viewer
- a generic RAG demo

## Strategic Upside

- reusable across many future corpora
- difficult to replicate well because trust requires provenance and review
- strong synergy with OpenClaw’s orchestration and graph infrastructure

## Strategic Risks

- credibility risk from overstated inference
- legal or political sensitivity around disputed interpretations
- entity-resolution mistakes causing false connections
- product confusion if OpenClaw and the incubated subsystem are not clearly
  separated

## Long-Term Defensibility

The moat is not access to one document dump. The moat is a disciplined system
for:

- preserving evidence
- extracting structured objects
- reconstructing decisions
- revising conclusions as new evidence arrives

## Adoption Risks

- if provenance is weak, serious users will not trust it
- if review is weak, the system will look like automated speculation
- if browsing is too raw, users will still feel overwhelmed

## Mitigations

- strict evidence classification
- stable citation anchors
- visible review and challenge states
- explicit uncertainty handling
